# AlmostCorrect

> Because correct logic isn’t always enough.

## About
This repository is a collection of challenging programming problems, my solutions, and the reasoning behind them.

The focus is not just on solving problems, but understanding:
- Why a solution works
- Why it *almost* failed
- The edge cases and hidden constraints involved

## Structure
- `problems/` → Documented problems and solutions
- `templates/` → Reusable problem-solving template

## Example Topics Covered
- Integer overflow
- Edge cases
- Algorithmic pitfalls
- Debugging insights

## Goal
To improve problem-solving skills by learning from mistakes, not just correct answers.
